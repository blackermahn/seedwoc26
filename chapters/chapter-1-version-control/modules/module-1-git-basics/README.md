# Module 1: Git Basics

## 📚 Module Overview

The foundation of all version control. Learn the essential Git commands and workflows that every developer uses daily.

## 🎯 Learning Objectives

By completing this module, you will:
- Initialize and manage Git repositories
- Create meaningful commits
- Understand Git's workflow
- Navigate project history
- Create and merge branches
- Work with remote repositories

## 📖 Module Outline

### Part 1: Introduction to Version Control
Understanding what Git is and why it matters

### Part 2: Core Git Workflows
Daily Git commands and best practices

### Part 3: Branching Fundamentals
Creating, managing, and merging branches

## ⏱️ Duration

- **Estimated time**: 1-2 weeks
- **Difficulty Level**: ⭐ Beginner

## 📝 Assignments

This module contains exercises covering:
1. Repository initialization
2. Commit creation
3. History viewing
4. Branch management
5. Remote operations

## ✅ Prerequisites

- Git installed on your system
- Basic command-line knowledge
- Text editor

## 🔗 Next Steps

After completing this module:
- Proceed to [Module 2: GitHub Collaboration](../module-2-github-collaboration)
- Apply Git in all future projects
- Practice with team repositories

## 💡 Module Tips

- **Type commands manually**: Don't copy-paste; build muscle memory
- **Experiment safely**: Create test repositories to try things
- **Read the output**: Git messages tell you what's happening
- **Ask questions**: Understanding the "why" matters more than the "how"

## 📞 Getting Help

- Check Git documentation: `git help <command>`
- Review lecture notes in each part
- Use the hints in exercise files
- Ask in discussions/issues
