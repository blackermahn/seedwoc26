# Chapter 8, Module 2: Database Integration and REST APIs

## Module Overview

This module covers database integration with Node.js and designing RESTful APIs. You'll work with both SQL and NoSQL databases, understand ORM/ODM libraries, and build complete API backends.

## Learning Objectives

By completing this module, you will be able to:
- Understand database fundamentals
- Connect to SQL databases (MySQL, PostgreSQL)
- Use ORMs (Sequelize, TypeORM)
- Connect to NoSQL databases (MongoDB)
- Use ODMs (Mongoose)
- Design RESTful APIs
- Handle CRUD operations
- Implement proper error handling
- Use database transactions
- Optimize database queries
- Implement pagination and filtering

## Curriculum Outline

### Part 1: SQL Databases and ORMs
- Database design and normalization
- Creating databases and tables
- SQL basics (SELECT, INSERT, UPDATE, DELETE)
- Connecting Node.js to SQL databases
- Using Sequelize ORM
- Models and associations
- Migrations
- Query optimization
- Connection pooling
- Duration: 3-3.5 hours
- Prerequisites: Chapter 8, Module 1

### Part 2: NoSQL Databases and ODMs
- NoSQL vs SQL concepts
- MongoDB basics and setup
- Document design
- Connecting with Mongoose
- Schema definition
- CRUD operations with Mongoose
- Population and relationships
- Indexing and optimization
- Transactions in MongoDB
- Duration: 2.5-3 hours
- Prerequisites: Part 1

### Part 3: RESTful API Design
- REST principles and constraints
- Designing RESTful endpoints
- Naming conventions
- HTTP methods and status codes
- Request validation
- Response formatting
- Pagination and sorting
- Filtering and search
- API versioning
- Duration: 2.5-3 hours
- Prerequisites: Part 1 or Part 2

### Part 4: Authentication and Security
- Authentication vs authorization
- Password hashing (bcrypt)
- JWT tokens
- Implementing user login/registration
- Route protection
- CORS handling
- SQL injection prevention
- Input validation and sanitization
- Rate limiting
- Duration: 2.5-3 hours
- Prerequisites: Part 3

## Practice Tips

1. **Design First**: Sketch API endpoints before coding
2. **Use Tools**: Postman for testing APIs thoroughly
3. **Database Visualization**: Use GUI tools to understand data structure
4. **Test Queries**: Test queries directly in database tools first
5. **Security Seriously**: Never skip validation and authentication
6. **Read Error Messages**: Database error messages are informative
